# Reference: Wibify Process

Source: https://wibify.agency/en

## Use As Inspiration

- Four-step process with numbering, concise descriptions, and practical tag lists.
- Clear progression from briefing to launch and care.
- Section hierarchy that makes the workflow feel predictable and premium.

## Do Not Copy

- Do not copy Wibify process names, wording, images, icons, or exact composition.
- Do not copy any studio-specific claims.

## Implementation Notes

- Use AIV-web process content from `docs/content.md`.
- Start as a theme pattern because this process is specific to the AIV-web homepage.
- Consider a reusable plugin block only if the same component becomes useful across multiple AIV projects.
- Use thin dividers and strong alignment rather than decorative icons.

## Performance Constraints

- Avoid large per-step images unless there is a real asset strategy.
- Keep interactions optional and CSS-driven.
- Make all steps visible and readable without JavaScript.
