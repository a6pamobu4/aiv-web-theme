# AIV-web

AIV-web is a standalone hybrid block-first WordPress theme for the AIV-web agency website.

This repository root is the WordPress theme root. Clone or copy this repository into:

```text
wp-content/themes/aiv-web-theme
```

This is a standalone client theme. It is not a shared parent theme and does not require a parent theme dependency.

## Goals

- Lightweight hybrid block-first architecture.
- No page builders, Bootstrap, Tailwind, jQuery, external fonts, or external icon libraries.
- Gutenberg-friendly and compatible with reusable custom blocks from a separate plugin.
- Suitable for the AIV-web agency website, landing pages, service pages, and later WooCommerce needs.
- Simple enough to maintain as a client-owned theme.

## Recommended Companion Plugins

- `aiv-blocks` for reusable custom Gutenberg blocks.
- `aiv-performance` for site-wide performance optimizations.

Performance changes that affect WordPress globally should live in a plugin, not in this theme.

## Styling Workflow

All custom styles are authored in SCSS.

- Edit source files in `assets/scss/`.
- Do not edit files in `assets/css/` manually.
- Compiled CSS is generated by npm scripts.
- WordPress enqueues compiled CSS from `assets/css/`.
- The theme `style.css` file is kept for the required WordPress theme header.

Install dependencies once:

```bash
npm install
```

Run watch mode during development:

```bash
npm run start
```

Create a production CSS build:

```bash
npm run build
```

The current theme styles compile from:

```text
assets/scss/base.scss   -> assets/css/base.css
assets/scss/editor.scss -> assets/css/editor.css
```

Reusable Gutenberg blocks should live in the companion `aiv-blocks` plugin. In block projects, keep `style.scss` and `editor.scss` inside each block folder and let the block build process compile CSS into the block build output.

## PHP Tooling

Install Composer dependencies:

```bash
composer install
```

Check PHP files against the configured WordPress Coding Standards rules:

```bash
composer run lint:php
```

Apply automatic PHPCS fixes:

```bash
composer run fix:php
```

## Client Theme Notes

This repository is already the AIV-web client theme. Add client-specific templates, patterns, SCSS, and block styles here while keeping reusable functionality in plugins when it should survive a future theme change.

## Theme vs Plugin

Belongs in the theme:

- Templates, template parts, and block patterns.
- Client-specific presentation styles.
- Theme supports, menus, image sizes, and frontend/editor assets.
- Minimal WooCommerce compatibility.

Belongs in a plugin:

- Custom post types and taxonomies.
- Custom Gutenberg blocks shared across websites.
- SEO, analytics, security, and caching behavior.
- Global performance changes such as disabling features or dequeuing WordPress assets.
- Business logic that should remain active if the theme changes.

## Development Notes

- Requires PHP 8.1+.
- Built for current stable WordPress and the block editor.
- Use current WordPress Developer Resources as the source of truth.
- Keep `functions.php` as a loader only.
- Add theme logic under `inc/`.
- Keep `theme.json` as the design system.
- Keep theme SCSS intentionally small and focused on the AIV-web website.
- Do not add aggressive performance hacks to the theme.
